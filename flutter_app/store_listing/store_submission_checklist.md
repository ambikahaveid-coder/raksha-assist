### Required Assets
- [x] App Icon 1024x1024 PNG (`assets/icons/app_icon.png`)

- [x] Description (in `app_store_listing.md`)

## 🚀 Step-by-Step Submission Process

### Google Play Store

1. **Create Developer Account**
   - Go to https://play.google.com/console
   - Sign in with Gmail
   - Pay ₹1,750 registration fee
   - Complete identity verification

2. **Create New App**
   - Click "Create app"
   - Enter app name: "Raksha Assist"
   - Select language: English (India)
   - Select "App" type
   - Select "Free" pricing

3. **Setup Store Listing**
   - Upload app icon
   - Upload feature graphic
   - Upload screenshots
   - Enter short description
   - Enter full description

4. **Complete Content Rating**
   - Answer questionnaire honestly
   - Medical apps typically get "Everyone" rating

5. **Set Up Pricing & Distribution**
   - Select countries (India)
   - Accept agreements

6. **Upload App Bundle**
   - Go to Release > Production
   - Create new release
   - Upload AAB file
   - Enter release notes
   - Review and rollout

7. **Submit for Review**
   - Complete all sections
   - Click "Send for review"
   - Wait 3-7 days

### Apple App Store

1. **Create Developer Account**
   - Go to https://developer.apple.com
   - Enroll in Apple Developer Program
   - Pay $99/year
   - Complete identity verification

2. **Create App in App Store Connect**
   - Go to https://appstoreconnect.apple.com
   - Click "My Apps" > "+"
   - Enter bundle ID: com.rakshaassist.app
   - Enter app name

3. **Setup App Information**
   - Upload app icon
   - Add screenshots for each device
   - Enter app description
   - Add keywords
   - Select categories

4. **Configure Privacy**
   - Add privacy policy URL
   - Complete data collection survey
   - Answer tracking questions

5. **Build & Upload**
   - Build iOS app with Xcode/Codemagic
   - Archive the build
   - Upload via Transporter or Xcode

6. **Submit for Review**
   - Select build
   - Complete review information
   - Submit for review
   - Wait 1-3 days

## 📝 Notes

### Common Rejection Reasons

**Google Play:**
- Missing privacy policy
- Inaccurate app description
- Broken functionality
- Requesting unnecessary permissions

**App Store:**
- Incomplete information
- Crashes or bugs
- Placeholder content
- Privacy policy issues

### Tips for Faster Approval
1. Test thoroughly before submitting
2. Provide clear app description
3. Include demo account if login required
4. Respond quickly to reviewer questions
5. Don't use medical/health claims without evidence
